Abstract

Sealed Positional Roman (SPR) is a novel symmetric encryption scheme that encodes data using Roman numeral representations with multiple cryptographic features including position-dependent rotation, prime-based ghosting, and configurable complexity scaling. This white paper presents comprehensive analysis of SPR's performance, security properties, quantum resistance capabilities, and competitive positioning against industry-standard algorithms.

Key Finding: SPR achieves 10-34× faster performance than AES-256/ChaCha20 while providing configurable quantum resistance up to 172 bits. However, lack of formal security proofs limits standardization potential to niche adoption scenarios.

Introduction

What is SPR?

SPR is a symmetric encryption scheme that transforms integers into position-dependent Roman numeral representations. Unlike traditional encryption that produces binary ciphertext, SPR generates human-readable character sequences that can be:

  • Spoken and transcribed
  • Memorized for short tokens
  • Transmitted over analog channels
  • QR-coded with reduced error rates
  • Debugged without hex editors

Core Innovation

SPR's unique value proposition is human-readable cryptographic output combined with exceptional performance. This makes it ideal for use cases where:

  • Users need to handle tokens manually (game codes, activation keys)
  • Visual confirmation is required (trading codes, session IDs)
  • Analog fallback is needed (voice transmission, paper backup)
  • Performance is critical (high-throughput encoding)
Clarification: Earlier analysis incorrectly classified SPR as "not encryption." SPR IS symmetric encryption with secret keys (radix, rotation, ghosting, etc.). The confusion arose from comparing SPR to Kyber (asymmetric key exchange) rather than AES/ChaCha20 (symmetric ciphers), which are the correct comparison class.

Key Components

SPR uses seven secret parameters as encryption keys:

  1. Radix: Private base (2-256+) - 4-8 bits entropy
  2. Rotation Key: Position permutation - log₂(n!) bits (e.g., 320 bits for n=32)
  3. Geometric Progression: Custom multipliers - variable entropy
  4. Ghosting Primes: Prime multiplier selection - log₂(primes) bits
  5. Starting Offset: Position shift - log₂(range) bits
  6. S-Box: Substitution tables - log₂(variants) bits
  7. Symbol Remap: Character substitution - 12.8 bits (7!)
Limitation: SPR has no formal security proofs. Security is based on Grover's algorithm bound (generic search complexity) rather than reduction to a hard mathematical problem like Learning With Errors (LWE) used by Kyber.

Standardization Prospects

Proposed Solution: SPR-512 Master Key + KDF

To address the "complex configuration" problem, we propose a single 512-bit master key with Key Derivation Function (KDF) that deterministically generates all seven SPR parameters: 

master_key (512 bits) → SHA-256-based KDF →
    • radix (16-256)
    • rotation_key (32 positions)
    • ghost_primes (0-255)
    • starting_offset (0-255)
    • sbox_seed
    • layers (1-3)
    • checksum_enable

Benefits

  • ✅ Single key format (like AES-256)
  • ✅ Compatible with key management systems (HSMs, vaults)
  • ✅ Simple API: spr_encrypt(data, master_key)
  • ✅ Easy to store/transmit (base64, hex)
  • ✅ No parameter mismatch errors

Honest Assessment

SPR is a technically sound, high-performance symmetric encryption scheme with a unique human-readable property. It achieves exceptional speed (10-34× faster than industry standards) and configurable quantum resistance (up to 172 bits). However, the lack of formal security proofs will likely prevent NIST/FIPS standardization, relegating SPR to niche adoption in non-compliance markets.

This outcome is not failure - many excellent algorithms thrive in niches. The key is owning the niche (human-readable crypto) rather than failing at universal adoption. SPR-LITE represents a genuinely unique solution for use cases requiring human-readable cryptographic output combined with high performance.

Bottom Line: Use SPR-LITE for what it's good at (human-readable tokens, game codes, session IDs) and use proven standards (AES-256, ChaCha20, Kyber) for everything else. Accept 7.5/10 as a realistic, achievable, and valuable outcome.
